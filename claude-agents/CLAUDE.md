# Xalq Insurance Digital OS - Claude Code Workspace

This directory is the Claude Code control plane for the Xalq Insurance Digital OS agent ecosystem.
Claude Code acts as the **developer domain brain** and the **meta-orchestrator** that
can drive the other three domains through MCP servers and the Python orchestrator.

Before broad work, also read the root workspace instructions:

- `../AGENTS.md`
- `../docs/RAMIN_OS_CONTEXT.md`
- `../SECURITY.md`
- `../services.json`

This directory is not a separate product; it is one control plane inside the
larger Ramin-OS system.

## Architecture (4 domains, 4 LLMs)

| Domain        | Runs on                          | Entry point                          |
|---------------|----------------------------------|--------------------------------------|
| Marketing     | CrewAI crew                      | `orchestrator/crews/marketing_crew.py` |
| Business      | CrewAI crew                      | `orchestrator/crews/business_crew.py`  |
| Developer     | Claude Code subagents (this dir) | subagents in `.claude/agents/`         |
| Personal Jarvis | isair/jarvis + Ollama          | `jarvis/` (voice front end)            |

LLM routing policy (see `orchestrator/router.py`):
- **Complex reasoning / code** -> Claude
- **Fast / cheap bulk** -> Groq (Llama 3.3 70B)
- **Free high-volume** -> Gemini 2.0 Flash
- **Private / offline** -> Ollama (local)

## MCP servers

Configured in `.claude/settings.json` (generated by `scripts/setup-mcp.ps1`):

| Server      | Purpose                                              |
|-------------|------------------------------------------------------|
| `n8n-mcp`   | Create and run n8n workflows                         |
| `filesystem`| Read/write inside the `ramin-os` project root        |
| `memory`    | Persistent knowledge graph across sessions           |
| `browser-use` | Drive a real browser for scraping and web tasks    |
| `elevenlabs`| Music + sound effects + TTS + voice cloning/dubbing  |
| `qdrant`    | Vector DB for RAG over project documents             |

## Subagents and skills

- `.claude/agents/`  - specialized subagents (populated by `scripts/install-agents.ps1`)
- `.claude/skills/`  - reusable skills (populated by the same script)
- `.claude/commands/`- slash commands; see `briefing.md` and `new-campaign.md`

## When to use which subagent

After running `install-agents.ps1`, prefer domain-specific subagents:
- Code review -> `code-reviewer` / language `*-pro` agents
- Security -> `security-auditor`, `penetration-tester`
- Bug fixing -> `debugger` / `bug-hunter`
- Docs -> `documentation` agents

Until the agents are installed, use the built-in `Explore`, `Plan`, and
`general-purpose` agents.

## Conventions

- Conversations with the user are in Azerbaijani; everything written to disk
  (code, configs, comments, docs) is in English.
- Never commit `.env`; only `.env.example` is tracked.
- Treat `data/` as runtime state - it is git-ignored.
