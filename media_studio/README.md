# Media Studio — the creative-director brain for Ramin-OS media production

> One sentence in → a directed, production-ready promo package out.
> `"mənə seedance 2.5 ilə 10 saniyəlik səyahət sığortası promo hazırla"`

Media Studio is the missing **brain** layer on top of the FLORA AI MCP that Codex
wired in. FLORA (and Higgsfield, Runway, etc.) already give you the *strong ML*:
Seedance, Kling, Runway, Sora, Veo video models exposed as an MCP tool. What
they do **not** give you is the marketer's instinct that turns a one-line
request into a professional brief. That instinct — frameworks, emotion arcs,
cinematic technique, model choice, brand DNA, compliance — is what Media Studio
adds. The pixels are still generated by FLORA's models; Media Studio is the
director standing behind the camera.

Implementation note: the Python package and service key remain `media_studio` for
backward compatibility. The product surface in the hub, docs, and UI is **Media
Studio**.

## Why this exists

People get "perfect" videos out of Higgsfield/FLORA not because of the model
alone, but because of a **good brief + reference + right model + iteration**.
That is the exact part most people do by hand and get wrong. Media Studio does it
automatically, at a senior-creative-director level, grounded in Xalq Sigorta's
brand and the Azerbaijani market.

## Architecture (three layers, we only build the top one)

| Layer | Owner | We build it? |
|---|---|---|
| Generation model (Seedance/Kling/Runway/Sora/Veo) | model labs | ❌ |
| Orchestration + MCP (FLORA / Higgsfield) | FLORA/Higgsfield | ❌ (we connect) |
| **Creative-director brain** (brief, model choice, compliance) | **Media Studio** | ✅ |

No LangChain. All LLM calls go through the repo's `llm_router` (LiteLLM,
free-first: Gemini → Groq → …). If a durable, human-approved generate→QA→publish
loop is ever needed, it plugs into the existing LangGraph spine, not a new
framework.

## Modules

- `knowledge.py` — durable creative intelligence: story **frameworks** (PAS,
  BAB, HSO, StoryBrand, AIDA), **emotional arcs**, a **cinematic technique
  library** (shots, camera moves, lighting, pacing, sound), and per-line
  **insurance playbooks** (travel, auto/KASKO, health, property). This is the
  "scenes that light up in a marketer's head", encoded and reusable.
- `models.py` — the FLORA video-model **catalog** + an **honest alias resolver**.
  Ask for "seedance 2.5" (which isn't in the catalog) and it maps to the closest
  real model *and surfaces a note* — it never silently invents.
- `director.py` — the **brain**: `parse_request` (deterministic intent) then
  `direct` (author a schema-valid brief via the LLM, with a strong deterministic
  fallback so it never fails).
- `pipeline.py` — the **orchestrator**: brief → compiled FLORA prompt (reuses
  `scripts/compile_generative_ad.py`) → storyboard + concept → **SVG storyboard
  board** (a real visual artifact) → cost-gated ready-to-fire command.
- `server.py` + `templates/studio.html` — the **front-end** (port 8870).

## Usage

CLI:

```powershell
python -m media_studio "mənə seedance 2.5 modeli ilə 10 saniyəlik səyahət sığortası üçün promo video hazırla"
python -m media_studio --no-llm "kling ilə 8 saniyəlik KASKO promosu"   # offline/deterministic
python -m media_studio --json "..."                                      # machine-readable
```

Front-end:

```powershell
uvicorn media_studio.server:app --port 8870
# then open http://127.0.0.1:8870
```

Media Studio is the primary workflow. After a package is created, it shows the
real-video stage plan and starts keyframes, animatic, film, and beat runs from
buttons in the browser. Paid FLORA stages require the on-screen approval gate
that includes the slug, stage, and current credit estimate; free/local stages
run without that paid confirmation.

Output lands in `output/media_studio/campaigns/<slug>/` (gitignored, regenerable):
`brief.json`, `prompts/compiled-flora-prompt.md`, `concept.md`, `storyboard.md`,
`storyboard-board.svg`, `ready-command.md`, `package.json`. The compiled prompt
plugs straight into `scripts/compile_generative_ad.py` and the FLORA MCP.

## The professional pipeline (keyframes-first, v2)

This is how high-end AI film work is actually done — script → boards →
**stills** → animatic → shoot → edit — and Media Studio now runs it end to end:

```powershell
python -m media_studio.generate <slug>                     # PLAN: every stage + its cost
python -m media_studio.generate <slug> --frames --confirm  # 1) keyframes, Seedream v3, ~cents
#    open frames/contact-sheet.html and pick the best still per beat →
python -m media_studio.generate <slug> --pick 1=2,3=1      # 2) selection (free)
python -m media_studio.generate <slug> --animatic          # 3) FREE Ken Burns animatic (local ffmpeg)
python -m media_studio.generate <slug> --beats --confirm   # 4) per-beat videos + trim/stitch master
python -m media_studio.generate <slug> --pro --confirm     # or all of it hands-free
```

Why it wins: the LOOK is approved at the still stage (cents), TIMING at the
animatic stage (free), and only then does video money get spent — with
beat-level redo instead of regenerate-everything. A **style bible** (lens,
light, grade, palette, composition with overlay negative space) plus a fixed
protagonist block is injected into every keyframe and every beat prompt, so all
shots live in one visual world. Storyboard beat timings are parsed and enforced
at stitch ("shoot long, trim short").

Engineering note: FLORA's raw `/generate` is prompt-only — image inputs exist
only in Techniques — so beat animation uses style-locked text-to-video with
continuity locks ("Shot N of a continuous sequence…", previous-shot linkage).
When a Technique with an `imageUrl` input exists, true image-to-video can slot
in without changing the flow.

## Real generation (the "işə sal" button)

FLORA OAuth is completed once via `mcp-remote` (token cached in `~/.mcp-auth`).
After that, `media_studio/flora_client.py` talks to the live FLORA MCP (tools
`execute` + `search_docs`) and `media_studio/generate.py` fires a real generation:

```powershell
python -m media_studio.generate <slug>            # PLAN only: model, cost, prompt — no spend
python -m media_studio.generate <slug> --confirm  # actually spend credits + generate + download
```

`--confirm` is the explicit spend authorization. Without it, the command prints
the model, the real credit cost, the exact params, and the cinematic prompt,
then stops at the cost gate. With it, it ensures a project, calls
`generations.create`, polls `generations.retrieve`, and downloads the `.mp4`
into the package folder. If the brief has no reference image, it picks the
text-to-video sibling of the recommended model automatically.

The model catalog in `models.py` is grounded against the live FLORA catalog
(167 video models, real `estimated_credits`) — e.g. "seedance 2.5" resolves to
**Seedance 2.0 Reference** (`i2v-seedance-2-0-reference-i2v-enhancor`, 1176 cr),
which genuinely exists; there is no literal "2.5".

## The cost gate (why it stops before spending)

Media Studio assembles **everything up to the single paid step** and then stops.
Per `config/agent_permissions.json` (`flora_ai_mcp` = draft_only), it:

- never spends FLORA credits automatically,
- never posts anything,
- shows a `run_cost`-first instruction and the one manual step (FLORA OAuth).

You say "işə sal", confirm the cost, and generation runs through FLORA's MCP.
Exact text/logo/price/CTA are then locked as **deterministic overlays** in Video
Studio — never rendered as generated pixels.

## Guarantees

- **Never fabricates** prices, dates, or model names. Unknowns become
  `[təsdiq gözlənir]` placeholders; unknown models get a surfaced substitution.
- **Never fails silently** — if the LLM is unavailable, the deterministic brain
  still ships a professional, schema-valid brief.
- **Deliverables in Azerbaijani**; code/config in English.

## AI UGC Campaign Pack

`--ugc` adds the Doruk-style AI influencer agency workflow as a native
Media Studio mode:

```powershell
python -m media_studio --ugc --no-llm "seedance 2.5 ile 10 saniyelik seyahat sigortasi ucun AI UGC influencer videosu"
```

It creates `ugc-pack/` inside the campaign folder:

- `persona.md` - synthetic creator card and consent/safety rules.
- `script-10s.md` - UGC hook/story/turn/CTA script.
- `voiceover.md` - Audio Studio route: rough TTS, natural clone, premium ElevenLabs gate.
- `video-prompt.md` - reference-still prompt, image-to-video prompt, text fallback, cost commands.
- `unit-economics.md` - video credits, voice/finish assumptions, margin formula.
- `landing-wireframe.md` - conversion page structure with payments disabled by default.
- `qa-checklist.md` - realism, voice, brand, legal, and synthetic-person checks.
- `resources-readiness.md` - safe local audit of FLORA, FFmpeg, Audio Studio, HF fallback,
  and approval gates without reading `.env` or pinging paid providers.
- `publish-dry-run.md` - final packaging path after a finished MP4 exists.

The mode is draft-only. It never spends credits, never calls payment APIs, and
never posts. The real video step still goes through `media_studio.generate` with
an explicit `--confirm` cost gate.

To answer "can this machine make Doruk-like real videos after approval?", run:

```powershell
python -m media_studio --resources
python -m media_studio.resources <slug>
```

The audit is offline by design: it verifies local wiring and dependencies, but
does not inspect credentials, OAuth caches, live credit balances, or provider
availability. Paid rendering still starts only from a human-run `--confirm`
command.
