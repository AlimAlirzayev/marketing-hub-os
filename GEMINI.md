# Ramin-OS Instructions for Gemini

This repository is one governed Ramin-OS product. Before planning or changing
anything, read and follow:

1. `AGENTS.md`
2. `docs/RAMIN_OS_CONTEXT.md`
3. `SECURITY.md`
4. `services.json`

## Mandatory User-Visible Delivery Gate

For every construction session, `docs/USER_VISIBLE_DELIVERY_STANDARD.md` is a
completion gate, not optional guidance.

Build the operator journey and UX/UI in parallel with the engine, place the
capability in its owning module and the unified Hub, expose real inputs and
truthful states, show the actual result, and validate the full flow from the
user side. A delivery without its applicable interface, Hub discovery, or
visible user-side proof must be reported as **partial**, never complete.

This requirement survives delegation: the lead session owns integration,
visual review, and the final operator handoff.
